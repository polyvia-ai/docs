The Index API handles vector embedding and indexing of parsed documents, making them searchable through semantic queries.

## Create Index

Trigger indexing for a parsed document to make it searchable.

### Endpoint

`POST /api/index`

### Request Body

<ParamField body="document_id" type="string" required>
  The ID of the document to index (must be already parsed)
</ParamField>

<ParamField body="embedding_model" type="string" default="text-embedding-3-large">
  The embedding model to use for vectorization
</ParamField>

<ParamField body="chunk_size" type="integer" default="512">
  Size of text chunks for embedding (in tokens)
</ParamField>

<ParamField body="chunk_overlap" type="integer" default="50">
  Overlap between consecutive chunks (in tokens)
</ParamField>

### Response

<ResponseField name="index_id" type="string">
  Unique identifier for the indexing job
</ResponseField>

<ResponseField name="document_id" type="string">
  Document being indexed
</ResponseField>

<ResponseField name="status" type="string">
  Indexing status: `pending`, `processing`, `completed`, `failed`
</ResponseField>

<ResponseField name="chunks_created" type="integer">
  Number of chunks created from the document
</ResponseField>

### Example

<CodeGroup>
```bash cURL
curl -X POST https://api.polyvia.ai/api/index \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "document_id": "doc_abc123xyz",
    "embedding_model": "text-embedding-3-large",
    "chunk_size": 512
  }'
```

```javascript Node.js
const response = await fetch('https://api.polyvia.ai/api/index', {
  method: 'POST',

    'Authorization': `Bearer ${apiKey}`,

  },
  body: JSON.stringify({
    document_id: 'doc_abc123xyz',

});
```

```python Python

    'https://api.polyvia.ai/api/index',

```
</CodeGroup>

<ResponseExample>

## Get Index Status

Check the status of an indexing job and retrieve statistics.

### Endpoint

`GET /api/index/{index_id}`

### Path Parameters

<ParamField path="index_id" type="string" required>
  The indexing job identifier
</ParamField>

### Response

<ResponseField name="index_id" type="string">
  Indexing job identifier
</ResponseField>

<ResponseField name="document_id" type="string">
  Document being indexed
</ResponseField>

<ResponseField name="status" type="string">
  Current status: `pending`, `processing`, `completed`, `failed`
</ResponseField>

<ResponseField name="chunks_created" type="integer">
  Total number of chunks created
</ResponseField>

<ResponseField name="vectors_embedded" type="integer">
  Number of vectors successfully embedded
</ResponseField>

<ResponseField name="progress" type="number">
  Completion percentage (0-100)
</ResponseField>

<ResponseField name="error" type="string | null">
  Error message if indexing failed
</ResponseField>

### Example

```bash
curl https://api.polyvia.ai/api/index/idx_xyz789 \
  -H "Authorization: Bearer YOUR_API_KEY"
```

<ResponseExample>
```json Success
{
  "index_id": "idx_xyz789",
  "document_id": "doc_abc123xyz",
  "status": "completed",
  "chunks_created": 45,
  "vectors_embedded": 45,
  "progress": 100,
  "error": null
}
```
</ResponseExample>

## Re-index Document

Re-index an existing document with new settings or after content updates.

### Endpoint

`PUT /api/index/{document_id}`

### Request Body

<ParamField body="force" type="boolean" default="false">
  Force re-indexing even if document hasn't changed
</ParamField>

<ParamField body="embedding_model" type="string">
  Update the embedding model
</ParamField>

### Example

```bash
curl -X PUT https://api.polyvia.ai/api/index/doc_abc123xyz \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{"force": true}'
```

## Chunking Strategies

<AccordionGroup>
<Accordion title="Fixed-size chunking">
  Default strategy that splits text into fixed-size chunks with overlap. Best for general-purpose indexing.
  
  **Recommended settings:**
  - Chunk size: 512 tokens
  - Overlap: 50 tokens
</Accordion>

<Accordion title="Semantic chunking">
  Splits text based on semantic boundaries (paragraphs, sections). Better for preserving context.
  
  <Info>
  Available in Pro and Enterprise plans.
  </Info>
</Accordion>
</AccordionGroup>

<Tip>
Smaller chunks provide more precise retrieval but may lose context. Larger chunks preserve context but may reduce precision. Experiment to find the optimal balance for your use case.
</Tip>
