The Retrieval API provides semantic search capabilities across your indexed documents, enabling precise information retrieval with citations and source tracking.

## Semantic Search

Perform semantic search across your entire knowledge base using natural language queries.

### Endpoint

`POST /api/retrieval/search`

### Request Body

<ParamField body="query" type="string" required>
  Natural language search query
</ParamField>

<ParamField body="limit" type="integer" default="10">
  Maximum number of results to return (1-100)
</ParamField>

<ParamField body="filters" type="object">
  Filter results by document metadata
  
  <Expandable title="Filter options">
    <ParamField body="document_ids" type="array">
      Limit search to specific documents
    </ParamField>
    
    <ParamField body="tags" type="array">
      Filter by document tags
    </ParamField>
    
    <ParamField body="date_range" type="object">
      Filter by document creation date
    </ParamField>
  </Expandable>
</ParamField>

<ParamField body="search_mode" type="string" default="hybrid">
  Search strategy: `semantic`, `keyword`, or `hybrid`
</ParamField>

<ParamField body="include_citations" type="boolean" default="true">
  Include source citations in results
</ParamField>

### Response

<ResponseField name="results" type="array">
  Array of search results ordered by relevance
  
  <Expandable title="Result object">
    <ResponseField name="chunk_id" type="string">
      Unique identifier for the text chunk
    </ResponseField>
    
    <ResponseField name="content" type="string">
      The relevant text content
    </ResponseField>
    
    <ResponseField name="score" type="number">
      Relevance score (0-1)
    </ResponseField>
    
    <ResponseField name="document_id" type="string">
      Source document identifier
    </ResponseField>
    
    <ResponseField name="document_name" type="string">
      Source document name
    </ResponseField>
    
    <ResponseField name="page_number" type="integer">
      Page number in source document (if applicable)
    </ResponseField>
    
    <ResponseField name="metadata" type="object">
      Additional metadata from the source document
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="total_results" type="integer">
  Total number of matching results
</ResponseField>

<ResponseField name="query_time_ms" type="integer">
  Query execution time in milliseconds
</ResponseField>

## Hybrid Search

Combine semantic and keyword search for optimal results.

### How It Works

Hybrid search leverages both:
- **Semantic search**: Understands meaning and context using vector embeddings
- **Keyword search**: Matches exact terms and phrases using BM25 algorithm

Results are ranked using a weighted combination of both scores.

### Configuration

<ParamField body="semantic_weight" type="number" default="0.7">
  Weight for semantic search (0-1)
</ParamField>

<ParamField body="keyword_weight" type="number" default="0.3">
  Weight for keyword search (0-1)
</ParamField>

<Note>
Weights must sum to 1.0
</Note>

### Example

```json
{
  "query": "machine learning algorithms",
  "search_mode": "hybrid",
  "semantic_weight": 0.6,
  "keyword_weight": 0.4
}
```

## Retrieve by Document

Retrieve specific chunks from a document with optional filtering.

### Endpoint

`GET /api/retrieval/document/{document_id}`

### Path Parameters

<ParamField path="document_id" type="string" required>
  Document identifier
</ParamField>

### Query Parameters

<ParamField query="page" type="integer">
  Retrieve chunks from a specific page
</ParamField>

<ParamField query="limit" type="integer" default="50">
  Maximum chunks to return
</ParamField>

### Example

```bash
curl "https://api.polyvia.ai/api/retrieval/document/doc_xyz789?page=5&limit=10" \
  -H "Authorization: Bearer YOUR_API_KEY"
```

## Advanced Filtering

### Filter by Date Range

```json
{
  "query": "quarterly reports",
  "filters": {
    "date_range": {
      "start": "2024-01-01",
      "end": "2024-03-31"
    }
  }
}
```

### Filter by Tags

```json
{
  "query": "compliance requirements",
  "filters": {
    "tags": ["legal", "compliance"]
  }
}
```

### Filter by Multiple Criteria

```json
{
  "query": "financial analysis",
  "filters": {
    "tags": ["finance"],
    "document_ids": ["doc_123", "doc_456"],
    "date_range": {
      "start": "2024-01-01"
    }
  }
}
```

## Ranking and Relevance

Results are ranked using multiple signals:

1. **Semantic similarity**: Cosine similarity between query and document embeddings
2. **Keyword match**: BM25 score for term frequency and document length
3. **Recency**: Newer documents receive a slight boost
4. **Source authority**: Documents with more citations rank higher

<Tip>
Use `semantic` mode for conceptual questions and `keyword` mode for exact matches. `hybrid` mode provides the best balance for most use cases.
</Tip>

## Rate Limits

- **Free tier**: 100 queries per day
- **Pro tier**: 1,000 queries per day  
- **Enterprise**: Unlimited queries

<Warning>
Large result sets (limit > 50) count as multiple queries against your quota.
</Warning>
